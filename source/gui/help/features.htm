Features

Here's what we've identified that Developer Playground can do:

  • Hook any running process (within security limitations)
  • Browse DLLs loaded by the process
  • Pick any DLL export for trace callbacks
  • Invoke OLEVIEW.EXE on any given module containing a type library
  • Invoke DEPENDS.EXE on any given module
  • Show Explorer properties for any given module
  • Search MSDN or Google for any given DLL export
  • Copy trace output to clipboard, or save it to file
  • Find a named export among running processes, using simple Find/Find Next
We like to think of Developer Playground as dumpbin.exe on steroids.
